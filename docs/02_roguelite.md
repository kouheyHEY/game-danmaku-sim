# 方針転換：ハクスラ／ローグライト化

> 既存の骨格（[01_skeleton.md](01_skeleton.md)）の上に「メタループ(Run)」を乗せる。
> 弾の世界(`World`)・`step()`・弾幕ライブラリはそのまま再利用する。

## ねらい

決まった（または半固定／ランダムな）敵の列を、**1体ずつ撃破して進む**。
撃破ごとに**3択から強化を1つ選び**、自機を育てて次の敵へ。自機HPが尽きたらラン終了。

- 反転（①②③④）は**中核ではなく時々のスパイス**（特殊な敵・イベント）として後から差し込む。
- 強化は**3択ドラフト**。

## 追加するドメイン概念（最小）

- **敵に耐久(HP)と撃破**：自弾が当たると `enemy.hp -= 武器の威力`、0で撃破して場から除去。
  既存の `bullet-hits-enemy` イベントを「ダメージ」に解釈し直すだけ（`step()` は無改修）。
- **自機HPと無敵時間**：敵弾被弾で `ship.hp--`。直後 `IFRAME` 秒は無敵（理不尽な連続被弾を防ぐ）。
- **Run（メタ層）**：`ロードアウト（HP・武器スペック）＋ 敵キュー ＋ 進行状態`。
  1戦ごとに現在のロードアウトと次の敵から `World` を生成する。

## レイヤと責務

```
Run（run/）            ← 状態機械：fighting → reward(3択) → 次へ／死亡=gameover／踏破=win
  └ buildEncounter      ← ロードアウト＋敵Spec から World を生成
  └ stepRun             ← step()のイベントを「敵HP減・自機HP減・撃破判定」に解決
World/step（domain/）   ← 弾の世界。決定論。無改修で流用
弾幕ライブラリ（pattern）← 敵弾にも自機武器にも使う（奇数/偶数/回転/ランダム/一方向）
RunRenderer（render/）  ← 戦闘描画＋HP/進行HUD＋3択リワード画面
```

- **強化はデータ**（`Upgrade`）。`apply(loadout)` で武器スペックやHPを書き換える。
  例：拡散（弾数+2）・連射UP・威力UP・弾速UP・最大HP+2・回復。
  自機武器を `WeaponSpec`（種類/弾数/拡散/速度/威力/連射）で持ち、`buildWeapon` で `Pattern` に変換 →
  強化がスペックを変え、次戦で武器が組み直される。
- **敵キューの生成関数を差し替えるだけ**で 固定／半固定／ランダム を切替可能（まずは段階的に強くなる固定列）。

## 反転スパイスの差し込み口（将来）

- 特定の敵を「逆弾幕(②)」「当ててはいけない(④)」エンカウントにする＝その戦闘だけ別ルールの解決関数/Objectiveを使う。
- 強化として反転系（グレイズ／キャッチ）を配る、なども可能。
